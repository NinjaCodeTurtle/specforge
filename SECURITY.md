# Security Policy

## Supported Versions

| Version | Supported |
|---|---|
| `main` | Yes |
| Historical tags and branches | No |

## Reporting a Vulnerability

Do not open a public issue for a suspected security problem.

If GitHub private vulnerability reporting is enabled for this repository, use
that channel. Otherwise, contact the maintainers privately through GitHub.

Please include:

- A short description of the issue
- The affected file or template
- Steps to reproduce
- The impact you believe it may have
- Any suggested remediation

## What to Expect

Maintainers will try to:

- Acknowledge receipt within 5 business days
- Triage the report and determine impact
- Coordinate a fix and update the repository

Because this repository primarily contains documentation and templates, the most
likely security issues are:

- Insecure example configurations or guidance in templates
- Misleading security architecture recommendations
- Template patterns that could lead to vulnerabilities if followed

## Disclosure Policy

We prefer to fix issues before discussing them publicly. After a fix is
available, maintainers may document the change in `CHANGELOG.md`.
